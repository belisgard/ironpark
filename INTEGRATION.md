# Интеграции: Hantico ↔ B2P

---

## Входящие (Hantico → B2P)

### A1. Создание партнёра

- **Тип интеграции**: входящая
- **Система-источник**: Hantico ERP
- **Набор данных**: внешний идентификатор партнёра, название компании, ИНН/ОГРН, юридический адрес, контактное лицо, телефон, электронная почта
- **Тип передачи**: HTTP

### A2. Выгрузка данных водителя

- **Тип интеграции**: входящая
- **Система-источник**: Hantico Customer Service
- **Набор данных**: внешний идентификатор водителя, фамилия имя отчество, телефон, дата рождения, серия и номер паспорта, номер водительского удостоверения, идентификатор автомобиля, идентификатор партнёра
- **Тип передачи**: Kafka

### A3. Справочник марок и моделей авто

- **Тип интеграции**: входящая
- **Система-источник**: Hantico ERP
- **Набор данных**: идентификатор марки, название марки, идентификатор модели, название модели, тип кузова, тип двигателя
- **Тип передачи**: Kafka

## Исходящие в Hantico (B2P → Hantico)

### B1. Список партнёров для ERP

- **Тип интеграции**: исходящая
- **Система-источник**: B2P
- **Набор данных**: идентификатор партнёра, внешний идентификатор партнёра, название компании, статус личного кабинета, контактные данные
- **Тип передачи**: HTTP

### B2. Список авто по населённому пункту

- **Тип интеграции**: исходящая
- **Система-источник**: B2P
- **Набор данных**: идентификатор автомобиля, марка, модель, государственный номер, статус аренды, населённый пункт
- **Тип передачи**: HTTP

### B3. Регистрация авто в диспетчерской

- **Тип интеграции**: исходящая
- **Система-источник**: B2P
- **Набор данных**: идентификатор автомобиля, государственный номер, марка, модель, VIN-номер, идентификатор диспетчерской → внешний идентификатор автомобиля
- **Тип передачи**: HTTP

### B4. Блокировка авто в диспетчерской

- **Тип интеграции**: исходящая
- **Система-источник**: B2P
- **Набор данных**: внешний идентификатор автомобиля, идентификатор диспетчерской, причина блокировки, дата и время операции
- **Тип передачи**: HTTP

### B5. Передача данных о штрафе

> Данные о штрафах поступают в B2P из API МВД (см. C1), после чего передаются в Hantico ERP для учёта.

- **Тип интеграции**: исходящая
- **Система-источник**: B2P
- **Набор данных**: идентификатор штрафа, идентификатор автомобиля, государственный номер, идентификатор водителя, дата нарушения, сумма штрафа, статус оплаты, дата оплаты
- **Тип передачи**: Kafka

### B6. Передача данных о списании аренды

- **Тип интеграции**: исходящая
- **Система-источник**: B2P
- **Набор данных**: идентификатор договора аренды, идентификатор водителя, идентификатор автомобиля, дата начала периода, дата окончания периода, сумма списания, дата списания, статус
- **Тип передачи**: Kafka

## Исходящие во внешние системы (B2P → External)

### C1. Получение штрафов

- **Тип интеграции**: исходящая
- **Система-источник**: B2P (запрос) / API МВД (ответ)
- **Набор данных**: гос. номер → дата нарушения, сумма штрафа, статус, скидка, срок скидки
- **Тип передачи**: HTTP
